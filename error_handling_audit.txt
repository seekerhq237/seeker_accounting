B. GENERIC POSTING ERROR MESSAGES (no guidance on how to fix)
================================================================================

Every posting action across the entire application catches ValidationError and
shows the raw exception string with no actionable guidance. When a user hits
"An active document sequence for sales_invoice must be configured before
posting", they have no idea WHERE to go or WHAT to do.

ONLY the payroll module (payroll_run_dialog.py, payroll_input_batch_dialog.py,
payroll_remittance_batch_dialog.py) currently detects "document sequence" errors
and offers navigation. All others show raw text.

--- SPECIFIC PREREQUISITE ERRORS THAT GIVE NO GUIDANCE ---

  Error: "An active document sequence for {type} must be configured before posting."
  Fix:   Go to Accounting Setup > Document Sequences
  Nav:   nav_ids.DOCUMENT_SEQUENCES
  User sees: Raw text. No button. No navigation hint.

  Error: "An AR control account mapping must be configured"
  Fix:   Go to Chart of Accounts > Account Role Mappings
  Nav:   No direct nav_id (dialog only, accessed from CoA page)
  User sees: Raw text.

  Error: "An AP control account mapping must be configured"
  Fix:   Same as AR
  User sees: Raw text.

  Error: "Invoice date must fall within an existing fiscal period."
  Fix:   Go to Accounting Setup > Fiscal Periods
  Nav:   nav_ids.FISCAL_PERIODS
  User sees: Raw text.

  Error: "Fiscal period '{code}' is locked. Posting is not allowed."
  Fix:   Go to Accounting Setup > Fiscal Periods and unlock/reopen
  Nav:   nav_ids.FISCAL_PERIODS
  User sees: Raw text.

  Error: "Item must have an inventory account configured"
  Fix:   Go to Inventory > Items and edit the item
  Nav:   nav_ids.ITEMS
  User sees: Raw text.

  Error: "The 'Payroll Payable' account role is not mapped..."
  Fix:   Go to Chart of Accounts > Account Role Mappings
  User sees: Raw text (but payroll post dialog now has quick-fix buttons).


================================================================================
C. MISSING PREREQUISITE VALIDATION
================================================================================

These are places where the user can get into a workflow that WILL fail because
a prerequisite is not configured, but there is no early warning. The failure
only surfaces when the user tries to save/post/submit — wasting their work.


--- DOCUMENT SEQUENCES ---

  Every posting service requires document sequences. None of the following
  pages check if sequences exist before allowing document creation:

  - Sales Invoices page (create invoice)
  - Customer Receipts page (create receipt)
  - Purchase Bills page (create bill)
  - Supplier Payments page (create payment)
  - Treasury Transactions page (create transaction)
  - Treasury Transfers page (create transfer)
  - Inventory Documents page (create document)
  - Depreciation Runs page (create run)
  - Payroll Calculation workspace (create run) [FIXED — now has nav guidance]
  - Payroll Input Batch dialog (create batch) [FIXED — now has nav guidance]
  - Payroll Remittance Batch dialog (create batch) [FIXED — now has nav guidance]

  Impact: User creates a draft document, fills out all fields, tries to post,
  THEN gets an error. All their input is wasted.


--- FISCAL PERIODS ---

  No page checks if a fiscal period exists or is open before allowing
  document creation in that date range. The user can:
  1. Create an invoice dated 2026-03-15
  2. Fill in all details
  3. Try to post
  4. Get "No fiscal period covers the posting date"
  5. Have to abandon the dialog, go create a fiscal period, come back


--- ACCOUNT ROLE MAPPINGS ---

  Sales/Purchases posting requires AR/AP control account role mappings.
  These are configured in a dialog accessible only from Chart of Accounts page
  or from certain module pages (Customers, Suppliers).
  There is no standalone nav_id for Account Role Mappings.

  Impact: Error message says "An AR control account mapping must be configured"
  but user has no idea where to find "Account Role Mappings" in the UI.


--- TAX CODE ACCOUNT MAPPINGS ---

  If invoices/bills include tax lines, the tax code must have an account mapped.
  Tax code account mappings are configured from the Tax Codes page only.

  Impact: User adds tax to invoice, tries to post, gets
  "Tax code 'VAT19.25' has no account mapped" — has to find Tax Codes page.


--- PAYROLL-SPECIFIC PREREQUISITES ---

  Creating employees requires:
  - Currencies (reference data — loaded from ReferenceDataService)
  - Departments (optional but combo is empty with no guidance)
  - Positions (optional but combo is empty with no guidance)

  Creating payroll components requires:
  - GL accounts for expense and liability mapping
  - If accounts not loaded, both combos are empty

  Posting payroll requires:
  - Payroll Payable account role mapping
  - Component expense/liability account mappings
  - Open fiscal period
  - Document sequence for journal_entry

  None of these are validated before the user starts the workflow.


--- INVENTORY ITEM PREREQUISITES ---

  Creating inventory items requires:
  - Units of measure (UOM combo can be empty)
  - Item categories (category combo can be empty)
  - GL accounts (all 4 account combos can be empty)

  item_dialog.py:270 has one good check:
    "Select a unit of measure. Create one in Inventory > Units of Measure first."
  But this is the ONLY such check in the entire codebase.


--- FIXED ASSET PREREQUISITES ---

  Creating assets requires:
  - Asset categories (combo can be empty)
  - Depreciation methods (combo can be empty)
  - Categories require GL account mappings

  Creating asset categories requires:
  - GL accounts (asset, accumulated depreciation, depreciation expense)
  - All 3 account combos can be empty with no feedback


--- TREASURY PREREQUISITES ---

  Creating transactions/transfers requires:
  - Financial accounts (created separately in Financial Accounts page)
  - Each financial account must have a GL account mapped
  - If no financial accounts exist, combos are empty with no guidance

  Financial account creation requires:
  - GL accounts (combo can be empty)
  - Currencies (combo can be empty)

E. COMPANY SETUP GAP
================================================================================

  When a user creates a new company, the following happens:

  1. company_service.py creates the company record
  2. company_seed_service.py optionally seeds chart of accounts

  What is NOT created automatically:
  - Fiscal periods
  - Journals
  - Document sequences (for ANY document type)
  - Account role mappings (AR, AP, payroll payable, etc.)
  - Tax code account mappings
  - Financial accounts
  - Payroll settings

  This means a new company has NONE of the infrastructure needed for any
  transactional workflow. The user must manually configure ~15+ items across
  6+ different pages before they can create and post their first invoice.

  There is no setup wizard, checklist, or guided onboarding.

  Typical new-user journey:
  1. Create company → success
  2. Navigate to Sales → create invoice → success (draft)
  3. Click "Post" → FAIL: "document sequence must be configured"
  4. User confused → where is this setting?
  5. Eventually finds Document Sequences page → creates sequence
  6. Tries to post again → FAIL: "AR control account not mapped"
  7. User confused → where are account role mappings?
  8. Eventually finds it under Chart of Accounts → maps AR
  9. Tries to post again → FAIL: "No fiscal period covers posting date"
  10. User finds Fiscal Periods → creates period
  11. Finally posts successfully

  This is a 10-step frustration loop that should be a guided setup.
