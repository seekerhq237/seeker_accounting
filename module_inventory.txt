================================================================================
  1. SYSTEM OVERVIEW
================================================================================

Seeker Accounting is a production-grade desktop accounting application for SMEs,
built with Python, PySide6 (Qt Widgets), SQLAlchemy, Alembic, and SQLite.

Product direction:
  - Multi-company support
  - OHADA-ready accounting foundation (SysCohada chart of accounts)
  - Full double-entry general ledger
  - Sales & receivables (invoices, receipts, allocations)
  - Purchases & payables (bills, payments, allocations)
  - Cash & bank management with reconciliation
  - Inventory with FIFO/weighted average costing
  - Fixed assets with 14 depreciation methods
  - Comprehensive payroll (Cameroon statutory compliance)
  - Contracts, projects, job costing, budgeting
  - Financial reporting (Trial Balance, GL, OHADA & IAS/IFRS statements)
  - Users, roles, and audit trail

Tech stack:
  - Python 3.11+
  - PySide6 / Qt Widgets (desktop UI)
  - SQLAlchemy ORM (persistence)
  - Alembic (migrations)
  - SQLite (default; portable to PostgreSQL)

  2. ARCHITECTURE & INFRASTRUCTURE
================================================================================

DEPENDENCY FLOW:
  UI → Service → Repository → Database Models/Session

LAYERING RULES:
  - UI collects input, displays results, calls services only
  - Services own validation, orchestration, posting, workflow state
  - Repositories own persistence and query shape only
  - Models define persistence structure

─── app/ ──────────────────────────────────────────────────────────────────────

  Bootstrap:
    - bootstrap_application() → Settings → DB init → Contexts → ServiceRegistry
    - Creates MainWindow with Sidebar + TopBar + WorkspaceHost

  Shell (MainWindow):
    - Left sidebar with navigation sections
    - Compact top context bar (active company, user)
    - Central workspace area (pages swap on navigation)

  Navigation:
    - NavigationService (QObject + signals)
    - 38 nav_ids defined (see Section 21)
    - Sidebar emits nav_id → WorkspaceHost swaps page

  Context:
    - AppContext: immutable application data
    - ActiveCompanyContext: signals on company change
    - SessionContext: DB session factories

  Dependency Injection:
    - ServiceRegistry: dataclass holding 100+ service instances
    - Factory functions organized by module
    - Single construction at bootstrap time

─── platform/ ─────────────────────────────────────────────────────────────────

  Exceptions (8 types):
    - ConfigurationError
    - ValidationError
    - NotFoundError
    - ConflictError
    - PermissionDeniedError
    - PeriodLockedError
    - PayrollSetupError
    - MissingAccountMappingError

  NumberingService:
    - issue_next_number(company_id, document_type_code) → str
    - Auto-generates document numbers with prefix/suffix/padding
    - Validates sequence exists and is active
    - Company-scoped

  Logging:
    - Dual output: console + rotating file (1MB, 3 backups)

─── shared/ ───────────────────────────────────────────────────────────────────

  UI Components:
    - BaseDialog: modal template (520×320, body_layout + button_box)
    - SearchableComboBox: live-search dropdown with value binding
    - ThemeManager: global theme controller (light/dark)
    - TokenManager: typography, spacing, radius, color tokens
    - Table helpers: compact dense table configuration
    - Status chips: consistent styling across modules

  Message Boxes:
    - show_info(parent, title, message)
    - show_warning(parent, title, message)
    - show_error(parent, title, message)
    - show_configuration_error(parent, title, message, action_label) → bool

  Utilities:
    - text_utils: humanize, truncate, coalesce

    - date_utils: format_date, format_month, parse_date

─── db/ ───────────────────────────────────────────────────────────────────────

  REPOSITORIES: None (bootstrap handles user/role setup)

  DTOs: None

  NOTE: The data model is complete and supports multi-user RBAC.
    The service layer exists but UI for user/role management is deferred.

  19. AUDIT MODULE
================================================================================

  Package: modules/audit/

  SCOPE:
    Append-only audit event trail. Records create/update/delete/post/approve
    events with actor context, entity references, and metadata. Immutable
    log — no UPDATE or DELETE workflows.

  MODELS (1):
    AuditEvent (audit_events): company_id, event_type_code, module_code,
      entity_type, entity_id, actor_user_id, event_description,
      metadata_json, created_at
    Indexed: (company_id, event_type_code), (company_id, module_code),
             (company_id, created_at)

  SERVICES (1):
    AuditService:
      - record_event(company_id, event_type, module, entity_type,
          entity_id, actor_user_id, description, metadata)
      - list_events(company_id, filters) → list[AuditEventDTO]
      - Guards listing with permission check

  REPOSITORIES (1):
    AuditEventRepository: add, list_by_company (with filtering)

  UI: Audit tab within PayrollOperationsWorkspace (uses AuditService)

  DTOs (2):
    AuditEventDTO, AuditEventFilterDTO
